---
name: post-strategy
description: Post the test strategy comment series on the work item / issue / PR. Reads the directory of Markdown files generated by test-guide-writer and posts each one as a separate comment in logical order, then back-fills the Table of Contents in Comment 1 with the captured comment URLs. Usage: /post-strategy [issue-number or work-item-id]
argument-hint: [issue-number or work-item-id]
disable-model-invocation: true
---

Post the test strategy comment series for $ARGUMENTS to the hosting platform.

Use the **orchestrator** agent's posting step. It will:

- Detect the platform from `git remote get-url origin`.
- Read the working directory written by `test-guide-writer` (default `${TMPDIR:-/tmp}/test-strategy-${ENTRY_TYPE}-${ENTRY_ID}/`) and its `index.json`.
- Post each comment file in order, capturing the resulting comment URL.
- PATCH Comment 1 to back-fill the Table of Contents with the captured URLs so every other comment is reachable in one click from the index.
- Cross-link the work item ↔ PR with a single pointer thread when applicable.
- Apply the `test-strategy-generated` label / tag (best-effort).

Follow the platform-specific instructions in the appropriate provider file:

- **GitHub** → `providers/github.md`
- **Azure DevOps** → `providers/azure-devops.md`
- **Generic** → `providers/generic.md` (no posting — substitutes back-link placeholders for offline reading and writes a combined `impact-analysis-report.md` to the working directory)

**No HTML file is produced and nothing is written to the repository working tree.**

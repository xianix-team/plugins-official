# Performance Analysis Report Template

This template defines the structure for the compiled performance analysis report. The orchestrator agent must follow this format exactly when compiling findings from the four analyzer sub-agents.

The report is **analysis-first**: it never directly modifies code. A separate optimization PR is opened only when fix-PR mode is explicitly requested (via `--fix-pr` or the `ai-dlc/pr/perf-optimize-fix` tag).

### How to get a fix PR

- Add the label **`ai-dlc/pr/perf-optimize-fix`** to this PR, or
- Re-run `/perf-optimize <pr-number> --fix-pr`.

A **separate** optimization PR will be opened with the Quick-wins applied. The source PR is never pushed to, and only findings explicitly classified as Quick-wins are included.

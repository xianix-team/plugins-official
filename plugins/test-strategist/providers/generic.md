# Provider: Generic / Unknown Platform

Use this provider when `git remote get-url origin` does not match GitHub or Azure DevOps — or as a fallback when API posting is not possible.

## Behaviour

In generic mode the test strategy is **not posted to any remote platform**. The Markdown comment files generated by `test-guide-writer` are the deliverable, and they are kept in a temporary directory **outside the repository working tree** so the run never pollutes git status.

This provider is intentionally read-only on the user's repo — no files are written to the repo, nothing is added to `.gitignore`, no commits are created.

---

## Output Layout

The `test-guide-writer` agent has already produced a directory of per-comment Markdown files plus an `index.json`:

```
${TMPDIR:-/tmp}/test-strategy-${ENTRY_TYPE}-${ENTRY_ID}/
├── index.json
├── 01-overview-and-focus.md
├── 02-risk-and-impact.md
├── 03-requirements-and-gaps.md
├── 04-tests-functional.md
├── 06-tests-security.md
├── 07-tests-privacy.md
├── 09-tests-resilience.md
├── 10-tests-compatibility.md
└── NN-coverage-and-signoff.md
```

This provider does two additional things:

1. **Substitutes the `${COMMENT_1_URL}` placeholder** in every comment file with the absolute path to `01-overview-and-focus.md`, so the back-link from later files works when reading offline.
2. **Concatenates** all comment files in `index.json` order into a single `impact-analysis-report.md` for one-shot offline review.

---

## Substitute the back-link placeholder

```bash
WORK_DIR="${1:-${TMPDIR:-/tmp}/test-strategy-${ENTRY_TYPE}-${ENTRY_ID}}"
INDEX="${WORK_DIR}/index.json"
test -f "${INDEX}" || { echo "test strategy index not found at ${INDEX}"; exit 1; }

COMMENT_1_PATH="${WORK_DIR}/$(jq -r '.comments[0].file' "${INDEX}")"

# Replace ${COMMENT_1_URL} placeholders with the local file path so back-links
# work when the .md files are read directly (e.g. in a Markdown viewer).
TOTAL=$(jq -r '.comments | length' "${INDEX}")
for k in $(seq 1 $((TOTAL - 1))); do
  REL_FILE=$(jq -r ".comments[$k].file" "${INDEX}")
  ABS_FILE="${WORK_DIR}/${REL_FILE}"
  TMP="${ABS_FILE}.tmp"
  sed "s|\${COMMENT_1_URL}|${COMMENT_1_PATH}|g" "${ABS_FILE}" > "${TMP}" \
    && mv "${TMP}" "${ABS_FILE}"
done
```

Comment 1's TOC also has placeholders; rewrite them to the local file paths:

```bash
TOC=$(jq -r '
  .comments
  | to_entries
  | map(
      "\(.key + 1). " +
      ( if .key == 0
        then "[\(.value.title)](./\(.value.file)) (this comment)"
        else "[\(.value.title)](./\(.value.file))"
        end )
    )
  | join("\n")
' "${INDEX}")

NEW_BODY=$(awk -v toc="${TOC}" '
  BEGIN { in_toc = 0 }
  /^## 📑 Contents/ { print; print ""; print toc; in_toc = 1; next }
  in_toc && /^> _The Contents links/ { in_toc = 0 }
  in_toc && /^---$/ { in_toc = 0 }
  !in_toc { print }
' "${COMMENT_1_PATH}")

echo "${NEW_BODY}" > "${COMMENT_1_PATH}"
```

## Build a single combined report

For convenience, concatenate the comment files into one Markdown document. This is purely a convenience artefact — the per-comment files remain the canonical deliverable for any platform that integrates with this plugin's index.

```bash
COMBINED="${WORK_DIR}/impact-analysis-report.md"
TITLE=$(jq -r '.work_item_title // "Impact Analysis & Test Strategy"' "${INDEX}")

{
  echo "# 🧪 Impact Analysis & Test Strategy — ${TITLE}"
  echo
  echo "_Generated by Test Strategist plugin (generic provider — no remote platform detected)._"
  echo

  for k in $(seq 0 $((TOTAL - 1))); do
    REL_FILE=$(jq -r ".comments[$k].file" "${INDEX}")
    cat "${WORK_DIR}/${REL_FILE}"
    echo
    echo "---"
    echo
  done
} > "${COMBINED}"
```

## Source of Code Changes

In generic mode, code changes are gathered from git against the base branch:

```bash
BASE=$(git symbolic-ref refs/remotes/origin/HEAD 2>/dev/null | sed 's|refs/remotes/origin/||' || echo "main")
git diff origin/${BASE}...HEAD
git diff --name-only origin/${BASE}...HEAD
git log --oneline origin/${BASE}..HEAD
```

The work item content must be provided by the user (pasted text or local file path).

---

## Output

On completion:

```
Test strategy generated for <entry-type> #<id>: <risk-level> — <N> test cases across <M> comments — written to <WORK_DIR>
Combined report (offline reading): <WORK_DIR>/impact-analysis-report.md
```

The `<WORK_DIR>` path is **outside the repository working tree** — it lives in `${TMPDIR:-/tmp}/test-strategy-...`. Nothing is written to the repo, and `git status` remains clean after the run.

---

## When to Use

This provider is the correct fallback for:

- Jira instances (no native API integration in this plugin)
- Self-hosted GitLab instances
- Bitbucket
- Any on-premises git server
- Local or offline runs where no remote API is available
- CI environments where only the comment files / combined Markdown are needed (e.g. uploaded as a build artefact by the surrounding pipeline)
